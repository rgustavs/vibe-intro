Chapter One
Chapter One · The Definition

What is an agent?

An AI agent is a software system that takes a goal, decides on its own how to approach it, and uses tools to take real actions until the job is done — or until it needs to ask for help.

Unlike a chatbot, which answers in one shot, an agent works in steps. It can search, read, write, call APIs, and check its own work along the way.

In one sentence
An agent = a language model + instructions + tools + a loop that lets it act.

Chapter Two
Chapter Two · The Brain

The LLM at the center.

A Large Language Model is the reasoning component that powers every agent. It reads text and writes text — fluently and at scale.

The LLM is what makes the agent feel intelligent: it can plan, summarize, decide which tool to use, and decide when it's done. But on its own, it can't do anything outside the conversation.

It doesn't know today's date, it can't read your email, and it forgets each conversation when it ends. Everything beyond reading and writing text has to be wired up around it — that's the rest of the agent.

Chapter Three
Chapter Three · Tokens

Everything is tokens.

Before the model sees your text, it's broken into tokens — small chunks of characters. The model reads tokens, thinks in tokens, and produces tokens.

Rough rule: 1 token ≈ 4 characters ≈ ¾ of an English word.

Example sentence, tokenized
The agent booked a meeting on Tuesday. → 8 tokens
01 — CONTEXT WINDOW
A finite working memory.
The model can only hold so many tokens at once — instructions, conversation, and any documents you give it all share the same space. Once it's full, older content falls out.
02 — COST & SPEED
You pay per token.
Every token in and every token out has a price and a time cost. Longer prompts and longer responses mean more money and more latency. Concise wins.
03 — DESIGN IMPACT
It shapes everything.
Deciding what goes into the context window — and what doesn't — is one of the central design questions in building any agent. "Just paste everything" has a real price.

Chapter Four
Chapter Four · The Components

What's around the model.

The LLM is one piece. The rest of the agent is the scaffolding that turns a smart text-generator into something that gets work done.

When teams talk about "building an agent," most of the work lives here — in the components around the model, not the model itself.

Five Building Blocks
01 / 05 System prompt The standing instructions. Tells the model who it is, what it's for, and how it should behave across every interaction.
02 / 05 Tools The actions the agent can take: search the web, query a database, send an email, run code, call an API.
03 / 05 Memory Notes and context from past sessions that get loaded back in when relevant, so the agent isn't starting from scratch every time.
04 / 05 Orchestration The control logic. Decides when to call the model again, when to use a tool, when to stop, and when to escalate to a human.
05 / 05 Guardrails Permissions, filters, and human checkpoints that keep the agent inside the lines of what it's allowed to do.

Chapter Five
Chapter Five · The Agent Loop

How it actually runs.

An agent doesn't answer in a single response. It works in a loop: think, act, observe, and repeat — until the job is done.

Each iteration uses tokens and carries forward what it learned from the last. This is what makes an agent autonomous: within the limits of its instructions and tools, it decides what to do next.

A good agent knows when to stop. A bad one loops forever.

Closing
Closing · What to Remember

Five things to carry with you.

The shortest possible version of everything we just covered. Use these to talk about agents with confidence.

The one-liner
"An agent is an LLM plus tools, instructions, and a loop — running until it has done the job."
01 An agent is more than a model — it's a system that can take actions.
02 The LLM is the reasoning engine. It reads and writes text, nothing more on its own.
03 Tokens are the unit of work — finite, metered, and they drive cost, speed, and design.
04 Most of the work lives in the scaffolding: prompt, tools, memory, orchestration, guardrails.
05 Agents work in a loop: think, act, observe, repeat — not a single response.
